# Phase Logs

This directory stores implementation logs by phase.

## Naming

Use one file per phase:

- `phase-0.md`
- `phase-1.md`
- `phase-2.md`

Continue the same pattern for later phases.

## Purpose

Each file should record what actually happened during implementation for that phase.

These logs are:

- factual
- phase-specific
- retrospective

They are not planning documents.

## Suggested structure

Each phase log should include:

- overview
- status
- goals for the phase
- what was implemented
- issues encountered
- verification performed
- deviations from plan
- what remains for the next phase

## Writing rules

- describe the final implemented state, not intentions
- record important technical decisions made during implementation
- include meaningful problems encountered and how they were resolved
- keep forward-looking work brief and limited to handoff context
